An abstract class in Java is a class declared with the `abstract` keyword that cannot be instantiated on its own and is meant to be extended by other classes. It can contain both abstract methods (methods without a body) and concrete methods (methods with implementation), along with fields and constructors. The main purpose of an abstract class is to define a common base structure for related classes while forcing subclasses to implement certain required behaviors. It’s especially useful when you want to share common code across multiple classes, enforce a consistent method structure, represent a general concept that should not exist independently, and support polymorphism by allowing different subclasses to be treated as the same base type.
=========================================================================
An interface in Java is a contract that defines what a class must do without telling how to do it. It declares methods that implementing classes are required to provide, separating structure from implementation. The main benefit of an interface is flexibility: different classes can implement the same interface and be treated as the same type, which enables polymorphism. This reduces dependency between parts of a program, makes code easier to extend or replace, and helps build clean, maintainable, and scalable applications.

A class provides implementation. It can have constructors, instance variables, and fully defined methods.
An interface defines a contract. It describes behavior but does not represent a concrete object.

A class cannot extend an interface because extends means inheritance of implementation. An interface does not provide full implementation or object state like a class does. Instead, a class must use implements, which means it agrees to follow the contract defined by the interface.

Multi-threading is the process of accomplising 2 or more steps simultaneously at once to enhace the internal CPU and functionality. The benefits of multi-threading is that it improves performance, keeps application responsive, handle multiple user request at a single time, and also perform background activities.

In Java, a thread goes through several stages during its life cycle. When a thread object is created using the `new Thread()` constructor but the `start()` method has not yet been called, it is in the New state. At this point, the thread exists only as an object in memory and is not eligible for CPU execution. Once `start()` is called, the thread moves to the Runnable state. In Java, Runnable includes both the state where a thread is ready to run and the state where it is actually running. The thread scheduler decides when the thread gets CPU time, so a Runnable thread may be actively executing or simply waiting for its turn.

The term Non-Runnable is not an official state in the `Thread.State` enum, but it is commonly used to describe threads that are alive yet temporarily unable to run. This includes threads that are blocked while waiting to acquire a lock, waiting indefinitely for another thread’s action (such as through `wait()` or `join()`), or sleeping for a specific duration using `Thread.sleep()`. In these situations, the thread is still active in the system but cannot proceed until a particular condition is met, such as receiving a notification, completing a timeout, or obtaining a lock.

A thread enters the Dead (or Terminated) state once its `run()` method finishes execution or if it terminates due to an uncaught exception. After reaching this state, the thread’s life cycle is complete and it cannot be restarted. Attempting to call `start()` again on a terminated thread results in an `IllegalThreadStateException`. In short, New means created but not started, Runnable means eligible to run or running, Non-Runnable refers to temporarily paused states like waiting or blocked, and Dead means execution has permanently finished.

Usually they occur due to external issues like file handling, database access, or network problems.
